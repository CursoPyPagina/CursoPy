Parámetros variables ¶

Supongamos que queremos crear una funcin que calcule la suma de tres nmeros:

def suma_tres(a=0, b=0, c=0) -> float:
    """Funcn que recibe tres parmetros
    y retorna su suma"""
    return a + b + c

Si queremos crear una funcin para calcular la suma de 4 nmeros, tendremos entonces, por ejemplo, que definir una funcin de 4 parmetros. Ahora, algo ms complicado, qu tal sino sabemos de manera especfica el total de nmeros que sumaremos, cmo implemenaremos entonces nuestra funcin sino sabemos cuntos parmetros configurar. Para ello est el concepto de parmetros variables.

Lo que podemos hacer con este tipo de parmetros es decirle a Python que no sabemos exactamente el nmero de parmetros a configurar en nuestra funcin, pero lo que s sabemos es lo que haremos con ellos. Por ejemplo, queremos definir una funcin que sume todos los valores ingresados en los parmetros, pero en este caso el nmero de parmetros puede ser de dos (para sumar dos nmeros), pueden ser tres (para sumar tres nmero), etctera. Para ello utilizaremos parmetros variables para que el nmero de parmetros en nuestra funcin pueda ser de cualquier nmero (por ejemplo dos parmetros, tres, cuatro, etctera), pero sin importar el nmero de parmetros, sumaremos los valores asignados a ellos. Con la idea anterior evitamos crear ms de una funcin para poder sumar dos nmeros tres ms.

Para implementar parmetros variables colocaremos dentro de los parntesis de nuestra funcin, el nombre que queremos asignarle a dicho parmetro. Luego, para especificar, justamente, que queremos que ese parmetro sea variable, colocaremos como prefijo un *. Por ejemplo, crearemos una funcin para calcular la suma de cualquier cantidad de nmeros:

In [19]:
# *valores: argumento variable
# puede representar 1, 2, 3, etcetera
# parametros
def sumas(*valores) -> float:
    """Funcin para calcular la suma de los
    valores ingresados. Los parmetros son
    variables y la funcin retorna el resultado
    de sumarlos.
    """
    # Almacenaremos en esta variable la suma
    # correspondiente de cada uno de los
    # valores ingresados en la funcion
    resultado = 0
    # Iteramos sobre los valores ingresados
    # en la funcion:
    for i in valores:
        # vamos sumando dicho valores
        # y el resultado quedara almacenado
        # en la variable resultado
        resultado += i
    # Retornamos la suma de los valores ingresado
    return resultado

# Probamos nuestra funcion:
print(f"Suma de 2 valores: {sumas(3,4)}")
print(f"Suma de 3 valores: {sumas(1,2,3)}")
print(f"Suma de 4 valores: {sumas(10, 40, 50, 4)}")

Más sobre Return ¶

Recordemos que toda funcin retorna un valor aunque no especifiquemos explcitamente ello. Cuando no incluimos el return en la definicin de una funcin, Python entiende que dicha funcin, en efecto, no regresa nada, pero para representar esta idea de la nada Python se vale de la clusula None. De tal manera, si no especificamos un retorno en una funcin, en automtico o por defecto dicha funcin retorna en realidad el "valor" de None. Por ejemplo

In [25]:
def mensaje(msj) -> None:
    """Funcin de un parmetro que no retorna nada.
    Esta funcin manda a imprimir el mensaje ingresado
    en el parmetro"""
    print(msj)
    
# Ponemos a prueba nuestra funcion
mensaje("Hola")

Hola

Donde, recordemos, la funcin mensaje() propiamente no retorna nada. De hecho, la idea de ver un resultado al ejecutar nuestras celdas de cdigo, por ejemplo cuando utilizamos la funcin print(), es muy distinto a producir o arrojar un resultado. Es decir, el hecho de que una funcin muestre algn resultado, es muy distinto a que dicha funcin produzca o arroje un resultado. As, tanto la funcin que definimos mensaje() como la funcin predefinida print() no retornan o arrojan algn resultado. En particular, la funcin mensaje() no retorna ningn resultado, donde, por defecto, implcitamente dicha funcin retornar el valor de None.

Lo anterior lo podemos ver de una manera ms visual si mandamos a imprimir el resultado de ejecutar la funcin mensaje(), o en otras palabras, podemos mandar a imprimir el valor None producido por la funcin mensaje():

Por otro lado, hasta el momento hemos visto que nuestras funciones slo retornan un valor, pero puede llegarse a presentar la necesidad de que una funcin retorne ms de un valor. Lo anterior se puede implementar de una manera muy sencilla si en el return colocamos los valores a retornar separados por comas. Por ejemplo

In [27]:
def sum_mult(*nums) -> tuple[float]:
    """Funcin de argumentos variables que retorna dos
    valores:
    - la suma de todos los nmeros ingresados
    - la multiplicacin de todos los nmeros ingresados"""
    # Variables auxiliares para almacenar las sumas
    # y las multiplicaciones
    suma = 0
    mult = 1
    # Calculos
    for i in nums:
        suma += i
    
    for i in nums:
        mult *= i
    # Retornamos dos valores
    # -> float
    # -> str
    # dos valores
    # -> ?
    return suma, mult

# ponemos a prueba nuestra funcion
print(sum_mult(1, 2.2, 3, 4.0, 5))

(15.2, 132.0)

de donde:

  • Cuando nuestras funciones retornan ms de un valor, Python agrupa de manera automtica, dichos valores de retorno, dentro de una tupla en el orden que colocamos estos valores. Por ejemplo, de la funcin anterior, al declarar return suma, mult se generar en automtico la tupla (<suma>, <mult>). Por ende, al ejecutar print(sum_mult(1, 2.2, 3, 4.0, 5)) vemos que la salida es la tupla (15.2, 132.0), siendo el valor de la primer entrada correspondiente a la suma (suma) de los parmetros ingresados y el valor de la segunda entrada correspondiente a la multiplicacin (mult) de los parmetros ingresados. En resumen, cuando retornemos ms de un resultado, stos se almacenarn en una tupla.

  • Una vez dicho lo anterior, tenemos entonces que sum_mult(<numeros>) es una tupla, en este caso, de dos entradas o de longitud 2, donde la posicin 0 corresponde a la suma de los nmeros y la posicin 2 a la multiplicacin. Por ejemplo

Recordemos que lo anterior es una buena prctica y opcional. Adems, representa el tipo de salida esperada, pero tambin puede ocurrir que dicha funcin no regrese exactamente lo que se menciona con esta instruccin.
