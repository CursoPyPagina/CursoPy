Función de pérdida ¶

  • Las funciones de prdida juegan un papel fundamental en el aprendizaje automtico.
  • Necesitamos de ellas para entrenar modelos porque nos dice qu tan bien nuestro modelo explica los datos.
  • Un valor alto de prdida indica que el ajuste del modelo es deficiente.
  • Por lo general, entrenamos modelos para minimizar la funcin de prdida. En algunos casos ser necesario, a su vez, maximizar una funcin en vez de minimizar, lo cual es sencillo de implementar simplemente colocando un signo de menos al inicio de la funcin y despus minimizamos.
  • Tenemos muchas opciones de funciones de prdida predefinidas en tensorflow. Algunas de las ms comunes son:
    • Error cuadrtico medio (MSE): tf.keras.losses.mse(). Penaliza fuertemente los valores atpicos y tiene una alta sensibilidad cerca del mnimo.
    • Error absoluto medio (MAE): tf.keras.losses.mae(). Escala linealmente con el tamao del error y tiene una baja sensibilidad cerca del mnimo.
    • Error Huber: tf.keras.losses.huber(). Es similar al MSE cerca de cero y similar a MAE lejos del cero.

Supongamos que utilizaremos MSE, para ello requeriremos de dos tensores de valores reales u "objetivos", y los valores pronsticados o "predicciones":

# Calculamos la perdida MSE
loss = tf.keras.losses.mse(targets, predictions)

lo que devolver un valor nico: promedio de las diferencias al cuadrado entre los valores reales y los predichos.

En muchos casos, el proceso de entrenamiento requerir que proporcionemos una funcin que acepta las variables y datos de nuestro modelo y devuelve una prdida. Por ejemplo, lo que haremos ser definir un modelo de regresin lineal $y=\beta_0+\beta_1x$ que toma:

  • $\beta_0$: el valor del intercepto;
  • $\beta_1$: el valor de la pendiente;
  • $x$: las caractersticas.
# Definicion de un modelo de regresion lineal
def linear_regression(intercept, slope = slope, features = features):
    # Devuelve las predicciones del modelo
    return intercept + features * slope

Lo que sigue despus es definir una funcin de prdida que acepta la pendiente, el intercepto, los objetivos y las caractersticas:

# Funcion de perdida calculada mediante mse
def loss_function(intercept, slope = slope, targets = targets, features = features):
    # Calculamos las predicciones para un modelo lineal
    predictions = linear_regression(intercept, slope)
    # Retornamos la perdida
    return tf.keras.losses.mse(targets, predictions)

As, podemos evaluar la ltima funcin definida para evaluar un conjunto de datos dado:

loss_function(intercept, slope, test_targets, test_features)

Entrenamiento por lotes ¶

El entrenamiento por lotes es utilizado para manejar grandes conjuntos de datos. En dichos casos en donde el conjunto de datos es excesivamente grande, entonces no podr realizarse el entrenamiento del modelo debido a que dicho conjunto no puede caber en la memoria. En su lugar, deberemos de dividir el conjuntos de datos por lotes de manera secuencial.

Un solo paso sobre todos los lotes se denomina poca y el proceso en s se denomina entrenamiento por lotes. Adems de aliviar las limitaciones de memoria, el entrenamiento por lotes tambin permitir que actualicemos ponderaciones del modelo y parmetros del optimizador despus de cada lote, en lugar de al final de la poca.

Adems, podremos cargar datos mediante pandas, pero hacindolo tambin por lotes. Por ejemplo, si tenemos un conjunto de datos de ms de 50 GB, es posible que deseemos evitar cargarlo todo a la vez. Podemos realizar la carga por lotes mediante el parmetro adicional de chunksize. As, lo que se har ser dividir el conjunto de datos en fragmentos y especificaremos el tamao de dichos trozos mediante chunksize, lo cual ahorra memoria computacional y mejora la eficiencia del cdigo.
