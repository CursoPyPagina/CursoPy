Tensorflow cuenta con operaciones de alto nivel, como tf.keras que permite saltarnos el lgebra lineal. Por ejemplo

# Definimos la informacion de entrada
inputs = tf.constant(data, tf.float32)

# Definimos la primera capa densa oculta.
dense1 = tf.keras.layers.Dense(10, activation='sigmoid')(inputs)

El primer argumento especfica el nmero de nodos salientes, en el ejemplo anterior hemos especficado 10 nodos salientes, y el segundo argumento es la funcin de activacin a utilizar. De forma predeterminada se incluir el sesgo. Adems, notamos que se han pasado las entradas como argumento para la primera capa densa.

Definamos otra capa densa

# Definimos la segunda capa densa oculta.
# Ahora con 5 nodos de salida
dense2 = tf.keras.layers.Dense(5, activation='sigmoid')(dense1)

donde ahora pasamos como argumento a la segunda capa densa, la informacin de la primer capa densa. Finalmente definimos la capa de salida

# Capa de salida
outputs = tf.keras.layers.Dense(1, activation='sigmoid')(dense2)

donde se pasa como argumento la informacin obtenida de la capa densa 2, y tendremos un nodo de salida.

# Informacion de entrada
inputs = tf.constant(data, tf.float32)

# Finalmente tenemos la capa de salida
outputs = tf.keras.activations.sigmoid(product2 + bias2)

donde siempre debe tenerse cuidado con las formas (shape) de los distintos componentes de la red neuronal para aplicar de manera correcta la multiplicacin matricial.

Funciones de activación ¶

Una capa oculta tpica consiste de dos operaciones. El primero realiza la multiplicacin de matrices, lo cual es una operacin lineal y el segundo aplica una funcn de activacin, lo cual es una operacin no lineal.

Veamos un ejemplo simple, donde asumimos que el peso sobre la edad es 1 y el peso sobre el monto de la factura es de 2

In [5]:
import numpy as np
import tensorflow as tf

# Proporcion de edad, esto es, edad / 1000
joven, viejo = 0.3, 0.6

# Proporcion del monto de las facturas, 
# esto es, monto  / 10000
factura_baja, factura_alta = 0.1, 0.5

# Aplicamos el paso de multiplicacion matricial para todas las
# posibles combinaciones de las caracteristicas

joven_baja = 1.0 * joven + 2.0 * factura_baja

viejo_baja = 1.0 * viejo + 2.0 * factura_baja

Dentro de las funciones de activacin tendremos algunas populares como:

  • Sigmoide: Se utiliza principalmente en la capa de salida de los problemas de clasificacin binaria.
  • relu: Por lo general se utilizar en todas las capas que no sean la capa de salida

  • Softmax: Se utiliza en la capa de salida en problemas de clasificacin con ms de dos clases. Las salidas de una funcin de activacin softmax se pueden interpretar como probabilidades de clase en problemas de clasificacin multiclase. Por ejemplo, si consideramos las siguientes probabilidades en la capa de salida:
0.4 --> clase1
0.9 --> clase2
0.1 --> clase3

tendremos entonces que la salida es ms probable que pertenezca a la clase 2.

Veamos un ejemplo de red neuronal, para un determinado problema de clasificacin multiclase, implementando las funciones de activacin:

# Alto nivel:

# Entrada
inputs = tf.constant(features, tf.float32)

# Capa densa 1
dense1 = tf.keras.layers.Dense(16, activation = 'relu')(inputs)

# Capa densa 2
dense2 = tf.keras.layers.Dense(8, activation = 'sigmoid')(dense1)

# Salida
outputs = tf.keras.layers.Dense(4, activation = 'softmax')(dense2)

Optimizadores ¶

En el entrenamiento de una red neuronal inicializamos de manera aletoria los pesos, punto de partida, despus medimos la prdida y luego intentamos pasar a obtener una prdida menor, esto es, deseamos minimizar la funcin de prdida de acuerdo al ajuste de los pesos. Para llevar a cabo este proceso utilizaremos el algoritmo del descenso del gradiente. En particular sabemos que el descenso del gradiente tiene peligro en quedars estancado en un mnimo local. Para resolver dicho problema utilizamos el descenso del gradiente estocstico.

Veremos un ejemplo en tensorflow para algunos optimizadores, de donde:

  • Optimizador SGD (descenso estocstico del gradiente): Lo utilizaremos mediante tf.keras.optimizers.SGD(), requerimos de una valor para la tasa de aprendizaje (determina qu tan rpido se ajustan los parmetros del modelo).

  • Optimizador RMS (optimizador de propagacin): Aplica diferentes tasas de aprendizaje a cada funcin (lo cual es sutil para problemas de alta dimensin). Adems tiene dos parmetros momentum y decay (al establecer un valor bajo para este parmetro evitar que el momentum se acumule durante largos perodos durante el entrenamiento). tf.keras.optimizers.RMSprop()

  • Optimizador ADAM: tf.keras.optimizers.Adam(), optimizador adaptativo. Proporciona mejoras adicionales y generalmente es una buena primera opcin de eleccin. Es similar a RMS al configurar el momentum para que decaiga ms rpido al reducir el parmetro beta1.

Veamos un ejemplo. Supongamos que las caractersticas y los pesos se han inicializado, referentes al problema sobre el crdito bancario con el cual hemos estado trabajando. Luego, definiremos un modelo que calcula las predicciones y una funcin de activacin sigmoidea

import tensorflow as tf

# Definimos una funcion del modelo
def model(bias, weigths, features=b_features):
    product = tf.matmul(features, weigths)
    return tf.keras.activations.sigmoid(product + bias)

Luego, definimos la funcin de prdida mediante binary_crossentropy, la cual es el estndar para los problemas de clasificacin binaria

# Calculamos los valores predictivos y la perdida
def loss_function(bias, weigths, targets=default, features=b_features):
    predictions = model(bias, weigths)
    return tf.keras.losses.binary_crossentropy(target, predictions)

Finalmente, definimos un optimizador utilizando RMS

# definicion del optimizador
op = tf.keras.optimizaer.RMSprop(learning_rate=0.01, momentum=0.9)
# minimizacion de la funcion de perdida, donde dicha funcion
# es de dos variables: bias y weights
opt.minimize(lambda: loss_function(bias, weights), var_list=[bias, weights])
